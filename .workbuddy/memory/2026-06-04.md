## 修复生产环境三功能全失效问题

**问题**：React Native (Expo Web) + Node.js 项目本地正常，上线后 AI 对话 / 实时消息 / 语音功能全部失效。

**根本原因**：
1. Nginx 只监听 HTTP 80，没有 HTTPS 443 配置 → 所有请求进不来
2. WebSocket location 没有 wss:// 支持 → Mixed Content 被浏览器拒绝
3. docker-compose 只映射了 80 端口，443 流量无法进入容器
4. 服务端 CORS 使用 `cors()` 无参数，credentials 请求被拦截
5. `server/index.js` 硬编码了真实 API Key（MIMO/BOCHA/ZHIPU），存在安全风险

**修复文件**：
- `nginx/nginx.conf`：新增 443 SSL server block + WSS 支持 + HTTP→HTTPS 强制跳转
- `docker-compose.yml`：新增 443:443 端口映射 + `/etc/ssl/rn-ai-chat` 证书目录挂载
- `server/index.js`：CORS 改读 `CORS_ORIGIN` 环境变量，移除所有硬编码 API Key
- `.env.example`：新增 `CORS_ORIGIN` 说明
- `docs/production-fix-guide.md`：完整操作手册（含证书配置、安全组、验证命令）

**服务器操作要点**：
1. 将 SSL 证书放到 `/etc/ssl/rn-ai-chat/{fullchain.pem,privkey.pem}`
2. 阿里云安全组放行 443 端口
3. `.env` 填写真实 MIMO_KEY / BOCHA_KEY / ZHIPU_API_KEY / CORS_ORIGIN
4. `docker compose down && docker compose build --no-cache nginx server && docker compose up -d`
